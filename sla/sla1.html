# 1. Concept and Example

# Concept

Supporting References are Seeded/Additional Data Elements in Transactions, that can be associated with Subledger Journal (Not the GL Journals)















# DFF Vs Supp Ref

  1. DFFs can store Additional Data Elements at Transaction level.

  2. Supporting References maintain Seeded/Additional Data Elements at SLA Journal Level (Based on Transaction level Data Elements).

  3. Unlike DFFs, Amount Balances for Supporting References can be maintained optionally.















# Types

Supporting References can be of two types

  1. With Balances
  2. Without Balances















# Examples

# E1 - Project Number

You need Revenue breakup by Projects

  1. You do not have Project Segment in COA and
  2. You have also not implemented Projects module

In such case you can implement a Supporting Reference called Project Number and easily get Revenue Account balance break up by Projects

This is example of With Balances Supporting Reference















# E2 - Customer

  1. You need Revenue break up by Customer but we don't have Customer Segment in COA

  2. In such case you can implement a Supporting Reference Customer and easily get Revenue break up by Customer

  3. This is example of With Balances Supporting Reference















# E3 - Insurance Agent

You need Insurance Expenses SLA Journals tagged by Insurance Agent

In such case you can implement a Supporting Reference Insurance Agent and tag it to SLA Journals for Payables Invoices

This is example of Without Balances Supporting Reference















# Important Note

If the information can be contained in the COA,

you should never create a Supporting Reference for the same.

As having the information as part of COA is a much stronger solution

than using the same as “Supporting Reference”.

However, you cannot keep all variables in the COA hence Supporting References can become handy at times.















# Sources

Supporting References can be sourced/derived from

  1. Seeded Header Columns E.g. Customer Name
  2. Seeded Line Columns E.g. Memo Line
  3. DFF Header Level E.g. Counter Number
  4. DFF Line Level E.g. Product Code















# Demo Examples

For Revenue break-up by Project (With Balances)

Product Code Product Name
HPPRB HP Probook 4540S
CR441 Corsair DDR4 RAM 3200

This will be sourced from Invoice Line Level DFF

Ac Manager Code Name
PJ101 Peter Jones
AN101 Angelica Paul
RJ101 Rajesh Patel

This will be sourced from Invoice Header Level DFF















# D3. Checkout Counter

Just to maintain Counter Number tagged to Revenue (Without Balances)

Counter Code Description
CC001 First Counter
CC005 Fifth Counter
CC008 Eighth Counter

This will be sourced from Invoice Header Level DFF















# D4. Customer Name

Just to maintain Counter Number tagged to Revenue (With Balances)

Customer Name
ABC Consulting

This will be sourced from Invoice Header Seeded Column















# D5. Line Description

Just to maintain Counter Number tagged to Revenue (With Balances)

Line Description
Class Fees

This will be sourced from Invoice Line Seeded Column
